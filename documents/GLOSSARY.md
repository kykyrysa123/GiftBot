# Telegram Бот для заказов — глоссарий ключевых терминов

## Пользователь (User)
Физическое лицо, взаимодействующее с ботом через Telegram. Может быть обычным пользователем, оформляющим заказы, или администратором, управляющим системой.

## Администратор (Admin)
Пользователь с расширенными правами доступа, который может просматривать все заказы, одобрять их, экспортировать данные в Excel и управлять базой данных.

## Состояние пользователя (User State)
Текущий этап взаимодействия пользователя с ботом. Управляется через ConcurrentHashMap и определяет, какие действия и сообщения доступны пользователю в данный момент.

## Заказ (Order)
Основная бизнес-сущность системы, содержащая информацию о товарах/услугах, которые пользователь хочет приобрести. Включает описание, ссылки, фотографии и контактные данные.

## Order Details (Детали заказа)
Текстовое поле в заказе, содержащее описание товаров, ссылки на продукты или текст "Заказ со скриншотом", если пользователь отправил только фотографии.

## Telegram File ID
Уникальный идентификатор файла (фотографии) в системе Telegram, который сохраняется в базе данных для последующего доступа к файлу без повторной загрузки.

## Валидация (Validation)
Процесс проверки корректности введенных пользователем данных:
- Телефон: формат +375...
- Username: наличие символа @
- ФИО: три слова
- Ссылки: проверка URL (требует улучшения)

## SQLite Database
Встроенная реляционная база данных, используемая для хранения заказов, информации о пользователях и file_id фотографий. Файл: orders.db

## Excel Generator (Генератор Excel)
Компонент системы, который создает Excel-файлы с информацией о заказах, включая гиперссылки на прикрепленные фотографии.

## Гиперссылка на фото (Photo Hyperlink)
Ссылка в Excel-файле, которая обеспечивает прямой доступ к фотографиям заказа через сохраненные file_id Telegram.

## ConcurrentHashMap
Структура данных Java, используемая для управления состояниями пользователей при параллельной обработке множества запросов.

## User Session (Сессия пользователя)
Временное состояние пользователя в процессе оформления заказа, которое сохраняется до завершения или отмены операции.

## Order Status (Статус заказа)
Текущее состояние заказа:
- Новый (необработанный)
- Одобренный (подтвержденный администратором)

## Telegram Bot API
Интерфейс программирования приложений Telegram, который обеспечивает взаимодействие бота с платформой Telegram для получения сообщений и отправки ответов.

## Apache POI
Библиотека Java, используемая для создания и обработки документов Microsoft Office, в частности для генерации Excel-файлов с отчетами по заказам.

## Inline Keyboard (Встроенная клавиатура)
Элемент интерфейса Telegram бота, отображающий кнопки непосредственно в чате ("Далее", "Отмена", "Оформить заказ").

## Notification Service (Сервис уведомлений)
Компонент системы, отвечающий за отправку уведомлений пользователям об изменении статуса заказа и администраторам о новых заказах.

## Order Processor (Обработчик заказов)
Компонент бизнес-логики, который управляет процессом оформления заказа, включая валидацию данных, сохранение в БД и управление состояниями пользователей.

## Photo Processing (Обработка фотографий)
Процесс получения, сохранения и управления фотографиями, прикрепленными к заказу, включая работу с file_id и создание ссылок в отчетах.

## State Management (Управление состояниями)
Механизм отслеживания текущего этапа взаимодействия каждого пользователя с ботом для обеспечения корректной последовательности действий.

## Batch Processing (Пакетная обработка)
Возможность системы обрабатывать несколько фотографий в одном заказе и генерировать отчеты по большому количеству заказов (100+).

## Data Integrity (Целостность данных)
Обеспечение сохранности и непротиворечивости данных заказов при параллельной работе multiple пользователей и перезапусках системы.

## Spring Boot Framework
Фреймворк Java, используемый для создания standalone приложения бота с автоматической конфигурацией и встроенным сервером.

## Maven
Инструмент для сборки проекта и управления зависимостями, используемый для компиляции и запуска приложения бота.

## Application Properties
Файл конфигурации (application.properties), содержащий настройки бота, включая telegram.bot.username и параметры базы данных.

## Business Logic (Бизнес-логика)
Набор правил и алгоритмов, определяющих работу системы: процесс оформления заказа, валидацию данных, генерацию отчетов и управление состояниями.

## Error Handling (Обработка ошибок)
Механизм управления исключительными ситуациями, такими как некорректный ввод данных, ошибки базы данных или проблемы с генерацией отчетов.
